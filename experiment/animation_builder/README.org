Rmagick側でmp4生成時のdelayの値によるbitrateやfpsの違い
|-------+--------------+-----+---------+----------|
| delay | bitrate kb/s | fps | 体感1手 | トータル |
|-------+--------------+-----+---------+----------|
|   100 |           50 |  25 | 1秒     | 5秒      |
|    50 |           89 |  25 | 0.5     |          |
|     1 |         1048 |  25 | 一瞬    |          |
|     0 |         1048 |  25 | 一瞬    | 0        |
|-------+--------------+-----+---------+----------|

100で作ったものをffmpegで変換
|-------+----+------+------------|
| delay | -r | 時間 | 備考       |
|-------+----+------+------------|
|   100 |  1 | 2:12 |            |
|   100 | 25 | 0:05 | ←同じ時間 |
|   100 | 30 | 0:04 |            |
|-------+----+------+------------|

ここで
ffmpeg -i _output.mp4 -f image2 %04d.png
すると大量に吐かれるので delay = 1 でいいのではないか？
